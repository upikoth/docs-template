## Идентификаторы NFR-{CAT}-{NNN}

Формат: **`NFR-{CAT}-{NNN}`**, где **CAT** — код категории из таблицы ниже (латиница, верхний регистр), **NNN** — 001, 002, … без пропусков внутри одного документа для данной CAT.

| Код CAT | Тематика документа                                                                                                             |
| ------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `PERF`  | Производительность и масштабирование.                                                                                          |
| `REL`   | Надёжность, доступность, отказоустойчивость, восстановление.                                                                   |
| `SEC`   | Безопасность, управление секретами, модель угроз на уровне требований.                                                         |
| `PRIV`  | Конфиденциальность персональных и чувствительных данных, минимизация.                                                          |
| `OBS`   | Наблюдаемость: логи, метрики, трассировки, аудит технических действий.                                                         |
| `OPS`   | Эксплуатация: релизы, откаты, конфигурация, резервирование сред.                                                               |
| `CMP`   | Соответствие нормам и внутренним политикам (без юридических формулировок — только требование к системе).                       |
| `UXQ`   | Качество взаимодействия с точки зрения **свойств** (доступность WCAG, согласованность терминов), не пользовательские сценарии. |
| `DATA`  | Данные: сроки хранения, бэкапы, миграции, качество данных на уровне политики.                                                  |
| `INT`   | Совместимость версий API, клиентов, форматов; ограничения интеграций **как платформенные свойства**.                           |
| `ENV`   | Ограничения среды и инфраструктуры (регион, изоляция, бюджет на задержку).                                                     |

Ниже каждый сохранённый пункт оформляется подразделом по образцу.
