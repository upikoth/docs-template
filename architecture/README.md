# Архитектура

Документация архитектурных решений продукта. Правила и процесс — в **`.cursor/rules/architecture.mdc`**.

## С чего начать

| Документ | Назначение |
| -------- | ---------- |
| [[system-overview\|system-overview]] | **Актуальная** схема компонентов и трассировка к FR/NFR |
| [[adr/README\|adr/README]] | Индекс всех ADR |
| [[changelog\|changelog]] | Хронология архитектурных изменений |

## Три слоя

1. **Обзор** (`system-overview.md`) — как устроено **сейчас**.
2. **ADR** (`adr/`) — **почему** принято каждое решение.
3. **Changelog** — **когда и зачем** менялась архитектура.

Первичное заполнение — навык Cursor **`architecture-init`**.

OpenAPI и код сервисов живут в каталогах `*-api/`, `*-web/`; здесь только решения и ссылки на контракты.
